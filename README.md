# AI Predictive Maintenance System

An end-to-end AI system that predicts industrial machine failure and explains what to do about it — combining a Random Forest classifier with an LLM explanation layer.

Most predictive maintenance systems stop at a risk score. This one tells you what is going wrong, which sensors are causing it, and what action to take.

**Live demo:** https://ai-predictive-maintenance-system.onrender.com  
*(Free tier — first load takes ~30 seconds)*

---

## What It Does

**Single prediction** — enter live sensor readings, get an instant risk assessment and a plain English AI explanation of what is happening and what to do next.

**Batch prediction** — upload a CSV or Excel file with multiple machine readings. Results appear instantly on a dashboard with summary cards (total / high / medium / low risk counts). The LLM only runs on high risk rows — fast and cost-efficient, exactly how you would design this in production.

**REST API** — integrate predictions into any existing plant monitoring system via a JSON endpoint.

**CSV export** — download the full batch results including AI explanations for high risk machines.

## The Dataset

Trained on the [AI4I 2020 Predictive Maintenance Dataset](https://archive.ics.uci.edu/dataset/601/ai4i+2020+predictive+maintenance+dataset) — 10,000 real industrial scenarios simulating a milling machine's operating conditions.

- **10,000** total scenarios
- **339** actual failures — a 3.39% failure rate
- **4** distinct failure modes: Tool Wear Failure (TWF), Heat Dissipation Failure (HDF), Power Failure (PWF), Overstrain Failure (OSF)

Each failure mode has a different sensor signature. Failures are never caused by a single sensor in isolation — it is always a specific combination of readings that matters.

---

## The Model

Random Forest classifier with `class_weight='balanced'` to handle the severe class imbalance (96.6% healthy, 3.4% failure).

**5 input features:**

| Feature | Unit | Valid Range |
|---|---|---|
| Air temperature | Kelvin | 290 – 305 |
| Process temperature | Kelvin | 305 – 315 |
| Rotational speed | rpm | 1168 – 2886 |
| Torque | Nm | 3.8 – 76.6 |
| Tool wear | minutes | 0 – 253 |

All inputs are validated server-side before reaching the model. Out-of-range values are rejected with a specific error message.

**Model performance** (stratified 80/20 split):

| Metric | No Failure | Failure |
|---|---|---|
| Precision | 0.99 | 0.79 |
| Recall | 0.99 | 0.82 |
| F1-score | 0.99 | 0.80 |

---

## The LLM Layer

After the model returns a prediction and probability, the system calls Groq (Llama 3.3 70B) with the sensor values and model output to generate a plain English explanation:

- What is likely going wrong
- Which sensor readings are most concerning and why
- What action the engineer should take

In batch mode, the LLM only runs on **High Risk** rows. Medium and Low risk rows return results instantly without an LLM call. This design decision keeps the system fast and cost-efficient at scale.

## Quickstart

### 1. Clone the repo

```bash
git clone https://github.com/OyewoleRasheed/predictive-maintenance.git
cd predictive-maintenance
```

### 2. Create a virtual environment

```bash
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Add your Groq API key

Create a `.env` file in the project root:

```
GROQ_API_KEY=your_key_here
```

Get a free key at [console.groq.com](https://console.groq.com).

### 5. Train the model

```bash
python train_model.py
```

This generates `machine_failure_model.pkl`.

### 6. Run the app

```bash
python app.py
```

Visit `http://localhost:5000` in your browser.

## API Reference

### `POST /api/predict`

Returns a prediction, failure probability, risk level, and LLM-generated explanation.

**Request**

```bash
curl -X POST https://ai-predictive-maintenance-system.onrender.com/api/predict \
  -H "Content-Type: application/json" \
  -d '{
    "air_temp": 298.7,
    "process_temp": 308.9,
    "rot_speed": 1408,
    "torque": 56.5,
    "tool_wear": 216
  }'
```

**Response**

```json
{
  "prediction": "Failure Likely",
  "failure_probability": 0.8731,
  "risk_level": "High",
  "explanation": "The machine is showing strong indicators of tool wear failure. The combination of high torque (56.5 Nm) and significant tool wear (216 minutes) is the primary concern — this interaction pattern is the leading cause of tool wear failure in this dataset. The engineer should immediately inspect and replace the cutting tool before the next production run."
}
```

### `GET /api/health`

```bash
curl https://ai-predictive-maintenance-system.onrender.com/api/health
# {"status": "ok", "model": "machine_failure_model.pkl"}
```

### `POST /predict_batch`

Upload a CSV or Excel file. Returns JSON with results for all rows and summary counts. High risk rows include LLM explanations.

### `POST /download_batch`

Accepts the batch results JSON and returns a downloadable CSV with all predictions and AI explanations.

## Known Failure Patterns

These sensor combinations reliably trigger high risk predictions:

| Failure Type | Key Pattern |
|---|---|
| Tool Wear Failure | High tool wear (>190 min) + High torque (>55 Nm) |
| Heat Dissipation Failure | Low rotational speed (<1250 rpm) + High torque (>65 Nm) |
| Power Failure | Very high speed (>2700 rpm) + Very low torque (<6 Nm) |
| Overstrain Failure | High torque (>70 Nm) + High tool wear (>170 min) |
